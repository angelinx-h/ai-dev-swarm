# Target Users

## Primary Audience

**Developers using AI agents with MCP servers**

### User Characteristics

- Familiar with MCP (Model Context Protocol) and AI agents
- Using AI code assistants in their development workflow
- Managing multiple MCP servers for various integrations
- Concerned about token costs and performance
- Comfortable with CLI tools and command-line workflows
- Using Node.js/TypeScript ecosystem

### User Context

- **Environment**: Local development machines (macOS, Linux, Windows)
- **Tools**: AI agents (Claude Code, Cline, etc.), MCP servers, package managers (npm/pnpm)
- **Skill Level**: Senior developers and technical users who understand agent concepts

### User Needs

- Reduce token consumption from MCP tool definitions
- Maintain full functionality of existing MCP tools
- Easy installation and setup (npm package)
- Transparent operation (works with existing MCP configs)
- Fast skill generation and updates
- Reliable HTTP bridge for runtime tool invocation

### User Expectations

- Tool "just works" without complex configuration
- Compatible with their existing MCP server setup
- Minimal performance overhead
- Clear documentation and examples
- Open source and community-maintained
